# EchoBlock — FAQ

**Q: Can one block have multiple triggers?**
A: Currently each block supports one trigger type. You can register the same block location under different IDs with different triggers as a workaround.

---

**Q: Does EchoBlock persist through restarts?**
A: Yes. All Echo Blocks are saved in `blocks.yml` and loaded on startup.

---

**Q: Can I use colour codes in messages?**
A: Yes. Use standard `&` colour codes (e.g., `&a` for green, `&c` for red).

---

**Q: What happens if a block is broken?**
A: If an Echo Block is broken, it will no longer trigger. Use `/echoblock remove` to clean up the entry from `blocks.yml`.

---

**Q: How do I report a bug?**
A: Contact **GalaxyModz** or **Frostbyte** via your support channel.
